简记二十二

2019-02-24
阅读次数 0

文章目录

  1. 关于公司
  2. 技术收获
  3. 对自己的期望

年前几次想做总结，都被一些琐事耽搁了，趁着即将二十三岁生日之际，就过去的近550天做个总结。

关于公司

一七年年底作为实习生加入河狸家，到现在毕业转正也大半年了。说实话真的非常庆幸当初的选择，进入一家符合自己脾气的公司是多么幸运。在杭州互联网疯狂尝试996的阶段，能够日常体验和领导较量谁下班早又怎能不有趣呢？

不过，如果从初次体验工作来说，并没有想象中的有趣。去年上半年大批技术人的离开，下半年组织架构的几次大变，对初入职场的人绝对是巨大的考验。无论如何，区别于北京总部的动荡，杭州技术团队稳定的人员构成，愉快的工作氛围，浓厚的分享氛围，还是相当OK的。

技术收获

大学求学期间，沉迷ACM竞赛无法自拔，大量的（老古董级别）算法训练占据了日常，而忽视了项目经验的积累。最早实习的几个月，不断地写出类似String val; if (val == "VALUE") {}的Java代码，从而每次匆忙上线的结果都是导师通宵打补丁。用C++写了几年算法之后，尝试Java按理说相当简单，但苦于不熟悉Java具体语法，同时也缺少对历史代码的认知，错误几乎贯穿了整个实习。

如果仅仅按部就班地工作，我想时至今日我也不会有写篇总结的想法。我所在服务端的大领导给了我相当大的帮助，对于我困惑的问题，他给出的回答都相当地客观。去年年中遇到了个事务提交异常的问题（具体描述为，在大事务里，被调用的方法抛出异常，但业务逻辑希望捕获该异常，并正常地提交事务。很遗憾，事务提交失败，被标记成只允许回滚）。最终我得到的答复是给了解决问题的两套方案，并可以给我一段时间去看Spring事务的源码，深入定位问题的本质原因。确实，因为这个问题，从看不懂Spring事务，到尝试了解Proxy，了解ASM，我几乎把架在JVM上的字节码模型都粗览了一遍（详见18年6-7月的几篇博客）。最初的问题也就迎刃而解了。

另一个值得炫耀的事，就数Linux Kernel的学习了。虽然还处在相当初级的阶段，但源代码的学习能够更准确地理解Linux操作系统完成各项任务所做的事情。诸如任务调度、进程内核态/用户态的切换、文件系统实现、网络实现等等，都不再是全盘托付OS，而是能够切实了解操作系统所扮演的角色。

RTFSC，只说Spring、ASM、Kernel这样的开源代码，我到还没有到认为这些是”Fucking Code”的阶段，无论是其中的设计还是实现，都完全是值得我去学习的。不过就公司的代码库来说，到确实是一大堆”Fucking Source Code”。只论技术收获，过去一年的收获完全是令人欣喜，值得炫耀的。

对自己的期望

如果说技术上收获颇丰，那么业务上的贫乏就是过去500多天最失败的经历了。虽然支撑过“时间库存”这么个基础级别的项目，负责其二方库的开发与维护，但是工作经验的匮乏总是被产品和其它技术牵着走，几乎是只负责技术实现而不再更多地考虑业务的应用场景了。未来的一年，只能希望借着负责的新项目来完成业务理解上的突破，不再局限在技术的一亩三分地。

当然，技术也是无法舍弃的。越是学习，越是觉得懂得太少。最近刚刚入门了事件驱动，才将将理解了所谓的NGINX高性能的缘由。希望未来的一年不要因为外务影响了探知的脚步。不要因为一知半解而沾沾自喜。

不要活成自己讨厌的样子
